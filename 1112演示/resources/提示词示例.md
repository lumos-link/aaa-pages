# Cursor提示词示例集合

## 一、项目初始化提示词

### 基础项目搭建
```
创建一个拍卖平台的高保真原型项目，使用HTML/CSS/JavaScript实现。

项目要求：
1. 创建标准的项目结构（HTML、CSS、JS分离）
2. 使用现代化的响应式设计
3. 主色调：#1a365d（深蓝色），辅助色：#e53e3e（红色）
4. 使用思源黑体或系统默认字体
5. 支持移动端、平板、桌面端

请创建基础的项目结构和首页框架。
```

## 二、页面生成提示词

### 首页/拍卖大厅
```
为拍卖平台创建首页，需要包含以下内容：

1. 顶部导航栏
   - Logo
   - 搜索框
   - 分类菜单
   - 用户登录/注册按钮

2. 轮播横幅区域
   - 3-5张轮播图
   - 自动播放
   - 指示器

3. 拍卖品列表
   - 使用卡片式布局
   - 每个卡片包含：
     * 商品图片（16:9比例，圆角）
     * 商品标题（最多2行，超出省略）
     * 当前最高价（红色高亮，大字号）
     * 剩余时间（倒计时，动态更新）
     * 出价按钮（悬停效果）
   - 响应式网格布局（移动端1列，平板2列，桌面3-4列）

4. 筛选和排序
   - 分类筛选
   - 价格排序
   - 时间排序

5. 分页或加载更多

设计要求：
- 现代化扁平设计
- 卡片有阴影和悬停效果
- 颜色使用之前定义的主色调
- 间距合理，视觉层次清晰
```

### 商品详情页
```
创建商品详情页，包含以下部分：

1. 商品图片区域（左侧或上方）
   - 主图大图展示
   - 缩略图列表
   - 支持点击切换

2. 商品信息区域（右侧或下方）
   - 商品标题
   - 商品描述
   - 起拍价
   - 当前最高价（突出显示）
   - 剩余时间（倒计时，大字号）
   - 出价次数
   - 出价按钮（大按钮，醒目）

3. 出价历史区域
   - 表格或列表展示
   - 显示出价人、出价金额、出价时间
   - 最新出价高亮

4. 出价表单（模态框）
   - 当前最高价显示
   - 输入框（最小出价提示）
   - 确认按钮
   - 取消按钮

5. 相关推荐
   - 相似商品推荐
   - 卡片式展示

设计要求：
- 信息层次清晰
- 重要信息突出
- 出价按钮醒目
- 响应式布局
```

### 用户中心
```
创建用户中心页面，包含以下功能模块：

1. 侧边栏导航
   - 个人信息
   - 我的拍卖
   - 我的出价
   - 我的收藏
   - 账户设置

3. 列表展示
   - 使用卡片或表格
   - 显示状态标识
   - 操作按钮（编辑、删除等）

设计要求：
- 清晰的导航结构
- 标签页切换流畅
- 列表信息完整
- 操作便捷
```

## 三、组件生成提示词

### 按钮组件
```
创建一个现代化的按钮组件系统，包含：

1. 主要按钮（Primary）
   - 深蓝色背景
   - 白色文字
   - 圆角
   - 悬停效果（背景变深）
   - 点击效果（轻微缩放）

2. 次要按钮（Secondary）
   - 白色背景
   - 深蓝色边框和文字
   - 悬停效果（背景变浅灰）

3. 危险按钮（Danger）
   - 红色背景
   - 白色文字

4. 禁用状态
   - 灰色背景
   - 不可点击

5. 加载状态
   - 显示加载动画
   - 禁用点击

使用CSS类实现，支持不同尺寸（小、中、大）。
```

### 卡片组件
```
创建一个通用的卡片组件，用于展示拍卖品信息：

1. 卡片结构
   - 图片区域（顶部）
   - 内容区域（中间）
   - 操作区域（底部）

2. 样式要求
   - 白色背景
   - 圆角（8px）
   - 阴影效果
   - 悬停时阴影加深，轻微上移

3. 响应式
   - 自适应宽度
   - 图片保持比例

4. 内容区域
   - 标题（加粗，最多2行）
   - 描述（灰色，最多3行，超出省略）
   - 价格（红色，大字号）
   - 时间（小字号，灰色）

使用CSS类实现，可复用。
```

### 表单组件

```
创建一套完整的表单组件系统：

1. 输入框
   - 标签（Label）

   - 错误提示（Error message）

2. 下拉选择

3. 文本域
   - 多行输入

4. 复选框和单选框
   - 自定义样式
   - 选中状态

5. 表单验证
   - 实时验证
   - 错误提示
   - 提交验证

设计要求：
- 统一的视觉风格
- 清晰的反馈
- 易于使用
```

## 四、交互功能提示词

### 倒计时功能
```
实现一个拍卖倒计时功能：

1. 显示格式：XX天XX小时XX分XX秒
2. 实时更新（每秒更新）
3. 时间到期后显示"已结束"
4. 时间少于1小时时，文字变红
5. 时间少于10分钟时，添加闪烁效果
```

```
2. 提交出价

4. 防重复提交
   - 提交后禁用按钮
   - 显示倒计时
```


## 提示词编写技巧

1. **明确具体**：描述要具体，避免模糊
2. **分步骤**：复杂需求分步骤实现
3. **提供示例**：可以描述期望的效果
4. **指定技术**：明确使用的技术栈
5. **迭代优化**：逐步完善，不要一次性要求太多

## 注意事项

- 每次对话保持上下文，在同一个对话中继续优化
- 如果AI理解有偏差，及时纠正
- 可以要求AI解释代码，帮助理解
- 生成的代码需要人工审查和测试
